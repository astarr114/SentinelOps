   486	## Splunk Integration Story
   487	
   488	SentinelOps is designed to work with **Splunk's full toolchain**:
   489	
   490	### Current (Demo Mode)
   491	Uses bundled sample data files (`src/data/`) through the `mockDataService.ts` abstraction layer. All primary flows work without any Splunk connection.
   492	
   493	### Splunk REST API Mode
   494	Set `SPLUNK_HOST` + `SPLUNK_TOKEN` in Settings → the abstraction layer routes queries to the Splunk REST API, returning real log data.
   495	
   496	### Splunk MCP Server Mode (v43 — Full Tool Connectivity)
   497	SentinelOps implements the **Splunk MCP Server 1.2** specification with complete tool support:
   498	
   499	**Connection flow:**
   500	1. Enter your Splunk base URL or ngrok HTTPS URL in Settings → Splunk MCP Server
   501	2. Enter your bearer token (or basic auth credentials)
   502	3. Click **Test MCP Connection** — SentinelOps runs `initialize` + `tools/list`
   503	4. Server name, version, and all available tools appear instantly
   504	5. Click **Save Configuration** — everything persists to the database
   505	
   506	**Supported operations:**
   507	| Tool | Purpose |
   508	|------|---------|
   509	| `splunk_run_query` | Run SPL search queries (primary) |
   510	| `splunk_get_indexes` | List all available indexes |
   511	| `splunk_get_index_info` | Inspect a specific index |
   512	| `splunk_get_info` | Server and version info |
   513	| `splunk_get_user_list` | List Splunk users |
   514	| `splunk_get_user_info` | Inspect a specific user |
   515	| `splunk_get_metadata` | Get sourcetypes/sources/hosts |
   516	| `splunk_get_kv_store_collections` | Browse KV store |
   517	| `splunk_get_knowledge_objects` | List saved searches/lookups |
   518	| `splunk_run_saved_search` | Execute saved searches by name |
   519	
   520	**Transport:**
   521	- **Endpoint**: `POST {base_url}/services/mcp` (auto-appended)
   522	- **Protocol**: JSON-RPC 2.0 Streamable HTTP
   523	- **Auth**: Bearer token or Basic auth
   524	- **Ngrok**: `ngrok-skip-browser-warning: true` auto-injected when URL contains "ngrok" (overrideable)
   525	- **HTTPS only** — plain HTTP connections are rejected
   526	
   527	### Splunk Hosted Models
   528	The `large-language-model` and `incident-followup` edge functions use the platform gateway pattern, pre-configured for Splunk Hosted Model compatibility via `X-Gateway-Authorization` header.
