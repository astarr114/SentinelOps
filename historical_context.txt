# SentinelOps — Hackathon Architecture Note

## Architecture Overview

SentinelOps separates **evidence gathering** from **AI reasoning** into two independently
configurable and independently attributed layers. Every analysis result carries explicit
metadata fields recording which layer was used.

```
┌─────────────────────────────────────────────────────────────┐
│                        SentinelOps                          │
│                                                             │
│  ┌───────────────────────────┐  ┌─────────────────────────┐ │

│  │                           │  │                         │ │
│  │  Local Splunk Enterprise  │  │  Gemini 2.5 Flash       │ │
│  │  ├─ REST API (:8089)      │  │  (via Medo gateway)     │ │
│  │  └─ MCP Server 1.2        │  │          OR             │ │
│  │     (/services/mcp)       │  │  Splunk Hosted Model    │ │
│  │                           │  │  (Splunk Cloud Platform │ │
│  └───────────────────────────┘  │   only — OpenAI compat) │ │
│                                 └─────────────────────────┘ │
│                                                             │
│  Every result records: evidenceSource + reasoningSource     │
└─────────────────────────────────────────────────────────────┘
```

---

## Layer A — Evidence Layer: Local Splunk Enterprise

### What it provides

| Capability              | Via REST API | Via MCP Server 1.2 |
|-------------------------|:-----------:|:------------------:|
| SPL search execution    | ✅           | ✅                  |
| Error log retrieval     | ✅           | ✅                  |
| Deployment event lookup | ✅           | ✅                  |
| Service metadata fetch  | ✅           | ✅                  |
| NL → SPL translation    | ❌           | ✅ (tool: splunk_run_query / splunk_run_search) |

### Configuration

```
REST base URL:   https://localhost:8089
REST token:      your Splunk API token (admin or search scope)
MCP endpoint:    https://localhost:8089/services/mcp
MCP token:       your Splunk MCP-scoped token (may differ from REST token)
MCP auth method: Bearer
SSL verify:      OFF for local dev (self-signed cert) — use ngrok for remote access
```

### ngrok for local Splunk

When running Splunk Enterprise locally and needing to expose it to the Supabase Edge Function
(which runs in Deno on Supabase's cloud), use ngrok:

```bash
ngrok http https://localhost:8089
```

Use the resulting `https://<subdomain>.ngrok-free.app` URL as both the REST base URL and the
MCP endpoint base URL. Supabase Edge Functions **cannot bypass SSL verification** — ngrok
provides a valid TLS certificate automatically.

## Layer B — Reasoning Layer: Provider Options

### Option 1 — Gemini 2.5 Flash (default, always available)

- Uses Google Gemini via the Medo gateway (API key injected by the platform).
- **Safe to claim at hackathon**: "AI reasoning is provided by Gemini 2.5 Flash."
- Clearly attributed in every analysis result as `reasoningSource: "gemini"`.

### Option 2 — Splunk Hosted Model (requires Splunk Cloud Platform)

> ⚠️ **Critical distinction**: Splunk Foundation Models / AI Assistant are available on
> **Splunk Cloud Platform only**. They are **not available on local Splunk Enterprise**.
> Configuring this option with a local Splunk Enterprise instance will fail — the endpoint
> does not exist on Enterprise.

- Requires: a real Splunk Cloud Platform instance with AI capabilities enabled.
- Endpoint format: `https://<your-instance>.splunkcloud.com/services/llm/v1/chat`
- Uses OpenAI-compatible `/chat/completions` format.
- Token: Splunk Cloud auth token with AI model access.
- Model name: the deployment ID returned by your Splunk Cloud AI endpoint.
- When active: clearly attributed as `reasoningSource: "splunk-hosted-model"` and
  `usedSplunkHostedModel: true` in every result.
- A "Test Hosted Model Inference" button in Settings verifies connectivity before use.

## Error Handling — No Silent Fallback

SentinelOps **never silently falls back from live to demo** when live credentials are configured:

1. If MCP is configured and fails → returns `error-mcp` with full error message; offers user choice to retry or switch to demo.
2. If REST is configured and fails → returns `error-rest` with full error message; offers user choice.
3. Demo mode is only used when: (a) no credentials configured, or (b) user explicitly clicks "Use demo mode" after a live failure.

This ensures judges can trust that a "LIVE" badge means genuinely live data.

## Hybrid Dual-Layer Architecture

SentinelOps separates evidence gathering from AI reasoning into two **independently configurable and independently attributed layers**:

```
┌─────────────────────────────────────────────────────────────┐
│                        SentinelOps                          │
│                                                             │
│  ┌───────────────────────────┐  ┌─────────────────────────┐ │
│  │   LAYER A — EVIDENCE      │  │  LAYER B — REASONING    │ │
│  │                           │  │                         │ │
│  │  Local Splunk Enterprise  │  │  Gemini 2.5 Flash       │ │
│  │  ├─ REST API (:8089)      │  │  (via Medo gateway)     │ │
│  │  └─ MCP Server 1.2        │  │          OR             │ │
│  │     (/services/mcp)       │  │  Splunk Hosted Model    │ │
│  │                           │  │  (Splunk Cloud Platform │ │
│  └───────────────────────────┘  │   — OpenAI compat API)  │ │
│                                 └─────────────────────────┘ │
│                                                             │
│  Every result records: evidenceSource + reasoningSource     │
└─────────────────────────────────────────────────────────────┘
```

**Key design principle**: Splunk Enterprise (local) provides the **evidence** (live log data, deployment events, service metadata). The **reasoning** (analysis brief, hypotheses, remediation narrative) is provided by a separately configured AI provider. Both are explicitly labelled in the UI and in every result payload.

> ⚠️ **Important distinction**: Splunk Foundation Models / AI Assistant are available on **Splunk Cloud Platform only**. They are **not available on local Splunk Enterprise**. Local Enterprise contributes evidence — not hosted AI inference.
